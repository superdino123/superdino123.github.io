C#中倾向叫集合，Java中倾向叫容器，本质是存储对象的方式

  • @SuppressWarnings注解及其参数表示只有有关”不受检查的异常”的警告信息应该被抑制

容器中泛型存在的意义

  • case
    • ArrayList 可以添加所有类型的对象; ArrayList 只能添加T及继承T的子类的对象
    • ArrayList遍历获取时取出来的是Object类型对象，需要进行类型转换; ArrayLst遍历获取对象取出来的是T对象
  • 意义
    • 添加对象时限制类型，类型错误可以在编译时就报错
    • 获取对象类型明确，不用强转类型

有哪些容器可以使用

  • Collection: 独立元素序列
    • List: 按照插入的顺序保存元素
      • ArrayList：有序存储
      • LinkedList：有序存储
    • Set: 不能有重复元素
      • HashSet:获取元素最快，无序存储
      • TreeSet：按照比较结果的升序存储
      • LinkedHashSet：按照被添加顺序存储
    • Queue: 按照排队规则来确定对象产生的顺序，只允许在容器的一端插入对象，并从另外一端移除对象
  • Map: 一组成对的”键值对”对象
    • HashMap: 提供了最快查找技术
    • TreeMap:按照比较结果的升序保存键
    • LinkedHashMap: 按照插入顺序保存键，与HashMap查找方式相同
