C#中倾向叫集合，Java中倾向叫容器，本质是存储对象的方式

优化是一个很棘手的问题，最好的策略就是置之不顾，直到你发现需要担心它了

  • @SuppressWarnings注解及其参数表示只有有关”不受检查的异常”的警告信息应该被抑制

容器中泛型存在的意义

  • case
    • ArrayList 可以添加所有类型的对象; ArrayList 只能添加T及继承T的子类的对象
    • ArrayList遍历获取时取出来的是Object类型对象，需要进行类型转换; ArrayLst遍历获取对象取出来的是T对象
  • 意义
    • 添加对象时限制类型，类型错误可以在编译时就报错
    • 获取对象类型明确，不用强转类型

有哪些容器可以使用

  • Collection: 独立元素序列
    • List: 按照插入的顺序保存元素
      • ArrayList：有序存储
      • LinkedList：有序存储
    • Set: 不能有重复元素
      • HashSet:获取元素最快，无序存储
      • TreeSet：按照比较结果的升序存储
      • LinkedHashSet：按照被添加顺序存储
    • Queue: 按照排队规则来确定对象产生的顺序，只允许在容器的一端插入对象，并从另外一端移除对象
  • Map: 一组成对的”键值对”对象
    • HashMap: 提供了最快查找技术
    • TreeMap:按照比较结果的升序保存键
    • LinkedHashMap: 按照插入顺序保存键，与HashMap查找方式相同

List使用的注意事项

  • ArrayList 便于随机访问元素，插入移除元素较慢
  • LinkedList 插入和删除元素代价较低，随机访问较慢
  • List的使用：https://github.com/superdino123/ThinkInJavaTest/blob/master/src/collections/ListFeatures.java

迭代器(设计模式)的应用

  • 迭代器解决了什么问题：
    • 在使用容器的过程中，不希望关心容器的类型。 如：原本对List编码，我想切换成Set，还需要从头修改代码
    • 将遍历序列的操作与序列底层的结构分离
  • 迭代器Iterator的限制：
    • 只能单向移动
    • api
      • iterator.next()
      • iterator.hasNext()
      • iterator.remove()
    • https://github.com/superdino123/ThinkInJavaTest/blob/master/src/collections/CrossContainerIteration.java
  • ListIterator与Iterator的区别
    • 只能用于各种List类的访问
    • 可以双向移动
    • https://github.com/superdino123/ThinkInJavaTest/blob/master/src/collections/ListIteration.java

Set的应用

  • Set不保存重复的元素
  • Set主要用于 查找元素 和 测试归属性
  • Set与Collection具有完全一样的接口，只是行为不同

对比Collection接口与Iterator方法

  • 实现Collection意味着需要提供iterator()方法
  • 两者的功能是基本相同的
  • https://github.com/superdino123/ThinkInJavaTest/blob/master/src/collections/InterfaceVsIterator.java
  • 生成Iterator是将队列与消费队列的方法连接在一起耦合度最小的方式，并且与实现Collection相比，它在序列类上所施加的约束也小得多

为什么foreach可以遍历所有Collection对象

  • 因为Iterable接口能够产生Iterator的iterator()方法，并且Iterator接口被foreach用来在序列中移动，因此可以遍历任何实现Iterable的类
  • 遍历系统变量：https://github.com/superdino123/ThinkInJavaTest/blob/master/src/collections/EnvironmentVariables.java
  • 适配器与Iterable中的应用：https://github.com/superdino123/ThinkInJavaTest/blob/master/src/collections/MultiIterableClass.java
  • Arrays.asList()产生的List对象的物理实现使用底层数组，如果列表有修改，原数组也会相应修改

注意

  • 新程序中不应该使用过时的Vector, Hashtable, Stack
