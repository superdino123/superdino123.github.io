什么是JavaBean

  • JavaBean是软件组件，具有属性，事件，方法
  • JavaBean的核心是内省，内省是分析Bean的过程，用于确定Bean的功能

什么是内省

  • 内省机制是可以发现并提供Bean的所有属性，事件，公有方法的机制，不能显示受保护方法和私有方法

如何添加属性

private double height;

public double getHeight(){
    return height;
}
public void setHeight(double h){
    height = h;
}

理解“JavaBean的配置设置可以保存在永久存储介质中，并且可以在以后恢复”

  • 意味JavaBean具有持久性，保存Bean当前状态到非易失性存储器的能力
  • Java类库提供的对象串行化能力，可以用于为Bean提供持久性
  • 只要实现java.io.Serializable接口使得串行化可以自动完成，Bean不需要采取其他操作
  • 如果Bean没有实现java.io.Serializable接口，则必须自己提供串行化。否则，容器不能保存组件的配置信息

通过CGI了解servlet

  • CGI: 浏览器通过CGI(公共网关接口)与Web服务器通信，允许单独的进程从HTTP请求读取数据，并将数据写入HTTP响应
    ​
  • Servlet: 不需要为每个客户端请求创建单独进程的CGI，因为是Java写的，所以还具备平台独立性质
  • 为什么Servlet不需要为每个客户端请求创建单独进程？servlet在Web服务器的地址空间内执行

从http请求 到 servlet(servlet的生命周期)

  • web浏览器输入url，生成http请求，发送到适当的服务器
  • http请求被Web服务器接受，服务器将这个请求映射到特定的servlet
  • 服务器调用servlet的init()方法，只有当servlet第一次被加载到内存中，才调用该方法。可以向servlet传递初始化参数，以便servlet配置自身
  • 服务器调用servlet的service()方法，处理http请求，servlet能够读取在http请求中提供的数据，为客户端定制HTTP请求
    • 处理http请求后，servlet会保留在服务器的地址空间中，并且可以用于处理从客户端接收到的其他HTTP请求，为每个http请求调用service()方法

写到这里，有此一问：用JavaBean写的程序就是servlet吗？还是servlet是一个Java已有的组件？

Tomcat是一个servlet服务器, 部署在Tomcat中的Java程序可以称为是一个servlet

  • 服务器从内存卸载servlet，调用destroy()方法

为什么servlet部署到tomcat就可以运行

  • Tomcat的lib中的servlet-api.jar包含构建servlet需要的类和接口
  • servlet编译后，必须使Tomcat能够找到，需要将servlet放置在tomcat中webapps目录的某个子目录下，并将servlet的名称输入到web.xml中
